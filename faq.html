FAQ

Practical answers for first-time nf-core/rnaseq users on HPC.

What is nf-core/rnaseq?

It is a community-maintained RNA-seq pipeline built with Nextflow. It runs common RNA-seq processing steps in a consistent, versioned, and reproducible way.

What is a samplesheet?

A samplesheet is a CSV file that tells the pipeline which FASTQ files belong to each sample. It usually includes sample name, FASTQ path, mate-pair path, and strandedness.

What is Apptainer?

Apptainer is a container system often used on HPC clusters. It lets nf-core run software in controlled containers without installing every tool manually in your account.

What is a dry run?

A dry run checks the command and planned workflow without launching the full analysis. It helps catch missing files, bad parameters, or configuration problems early.

Why should I check HPC resources first?

RNA-seq runs can use a lot of disk space, memory, and scheduler jobs. Checking resources first helps avoid failed jobs, full disks, and long waits caused by incorrect queue settings.

How much storage do I need?

It depends on sample number, read depth, aligner, and whether intermediate files are kept. As a practical starting point, reserve several times the size of your compressed FASTQ files, and more if keeping BAM files or genome indices.

Can I resume a failed run?

Yes. Nextflow supports `-resume`. Keep the `work/` directory and run the same command with `-resume`. If you changed important parameters, check carefully because some tasks may need to rerun.

What files should I keep after the run?

Keep count or quantification outputs, `multiqc_report.html`, `pipeline_info/`, `software_versions.yml`, `configs/configure.yaml`, the samplesheet, configs, scripts, and reference metadata. Keep BAM files only if you need them and have enough storage.

What is MultiQC?

MultiQC is an HTML report that gathers QC metrics from many pipeline tools. It is the easiest place to review sample quality, mapping rates, duplication, trimming, and other run-level summaries.

What are pipeline_info files?

They are metadata files created by the pipeline. They record parameters, software versions, execution timeline, resource use, and reports that help reproduce or audit the run.

What should I do if STAR is killed?

Check the task log and `.nextflow.log`. STAR often fails because of memory limits, genome index size, or walltime. Increase memory/time for STAR tasks, use the right queue, or ask the HPC team about recommended resources for your genome.

What should I do if I run out of storage?

Stop the run, check which folders are largest, and avoid deleting `work/` if you want to resume. Move or archive unnecessary intermediate files, request more quota, or rerun with options that keep fewer large intermediates.
