Scripts involved

          
configure.sh

          
configure.sh auto-generates configs/configure.yaml with local paths, HPC destination, aligner choice, profile, work directory, output directory, cache path, and estimated memory, CPU, and walltime settings. It scans project metadata, reference files, and samplesheet information to estimate resources for the project.

          
Example configs/configure.yaml fields:

          
aligner: star_salmon
fastq_dir: input/fastq
samplesheet: input/samplesheet.csv
reference: input/reference/genome.fa
annotation: input/annotation/genes.gtf
species: oat
genome_size_gb: 12
sample_count: 30
hpc_user: your_username
hpc_host: your.cluster.edu
hpc_project_dir: /scratchdata1/your_project/rnaseq_projects/my_project
outdir: results
workdir: work
cache_dir: /scratch/your_username/apptainer_cache
memory: 64G
cpu: 16
walltime: 24:00:00

            

              
Notes

              
    
            
  • The script writes configs/configure.yaml automatically.
    • 
            
  • configure.sh scans project files to infer species, genome size, and sample count, then estimates memory, CPU, and walltime.
    • 
            
  • Open configs/configure.yaml after generation and quickly check the paths.
    • 
            
  • Only override memory, CPU, walltime, scratch, and profile values if your HPC needs different settings.
    •

☑ Allow automatic fixes and resume during validation

          
Safe infrastructure and resource fixes may be applied automatically. Biological settings are never changed automatically.

          
Scripts involved

          

            

              
Smoke Test

              
smoke_run.sh

              
Purpose: Validate the HPC environment using a tiny built-in test dataset.

              
    
                
  • Check Nextflow, Apptainer/Singularity, container downloads, cache configuration, permissions, modules, work directory, and scratch directory
    • 
                
  • Auto-fix container download failures, missing cache directory, bad cache path, permission issue, module issue, and common Nextflow problems
    • 
                
  • Behaviour: Detect → Auto-fix → Resume
    • 
              

              
Output: reports/smoke_test_report.md

              
How to run:

              

            

            

              
One Sample Run

              
one_sample_run.sh

              
Purpose: Validate the current project configuration and estimate real resource requirements.

              
Workflow: Dry Run → Run one real sample.

              
Default sample: Largest sample, because resource validation should detect STAR memory issues, BEDTOOLS memory issues, walltime limits, scratch/work directory problems, disk quota issues, and bottlenecks.

              
    
                
  • Advanced options: largest sample, smallest sample, or user-selected sample
    • 
                
  • Auto-fix STAR memory failure, CPU/thread mismatch, BEDTOOLS memory failure, walltime issues, quota or TMPDIR issues, scratch recommendations, and resumable task failures
    • 
              

              
Output: reports/one_sample_validation_report.md

              
How to run:

              

            

          

            

              
Notes

              
    
            
  • Review reports/smoke_test_report.md.
    • 
            
  • Use the largest sample by default when the goal is resource validation.
    • 
            
  • Review reports/one_sample_validation_report.md before launching the full run.
    • 
            
  • Never auto-change: reference genome, annotation, strandedness, aligner choice, metadata, sample grouping, or experimental design.
    •
